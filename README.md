# OrgCoreMonitorCQA.Sprint4

## 📌 Projeto

Este repositório contém os artefatos da Sprint 4 da disciplina de Compliance & Quality Assurance, referentes ao projeto CoreMonitor.

O CoreMonitor é uma plataforma contábil-financeira integrada com IoT, desenvolvida para centralizar dados operacionais e financeiros, permitindo o gerenciamento de clientes, contas contábeis, centros de custo, registros contábeis, compras e vendas.

---

## 🎯 Objetivo da Entrega

A entrega tem como objetivo apresentar o plano de testes manuais da aplicação e os testes automatizados de API, conforme solicitado na Sprint 4.

A entrega contempla:

- Plano de testes manuais em PDF;
- Testes automatizados utilizando Bruno;
- Evidências de execução;
- Link do Azure Boards;
- Link do vídeo demonstrativo.

---

## 📄 Plano de Testes Manuais

O plano de testes manuais está disponível em:

`Plano_Testes_CoreMonitor_CQA_Sprint4.pdf`

O documento contempla:

- Testes planejados;
- Dados de entrada;
- Dados de saída esperados;
- Procedimentos de execução;
- Validação das principais funcionalidades do sistema.

---

## 🧪 Testes Automatizados

Os testes automatizados foram criados utilizando a ferramenta Bruno.

A coleção de testes está localizada em:

`/bruno/CoreMonitor_API_Tests/`

Foram considerados quatro cenários principais:

1. Teste de sucesso com dados válidos;
2. Teste de erro com campo obrigatório ausente;
3. Teste de erro com valor inválido;
4. Teste de erro com registro inexistente.

---

## ▶️ Como Executar os Testes no Bruno

1. Abrir o Bruno;
2. Importar a coleção localizada na pasta `/bruno`;
3. Configurar a URL base da API, se necessário;
4. Executar as requisições;
5. Validar os status code e os retornos esperados.
